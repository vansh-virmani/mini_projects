default_app_config = 'accounts.AccountsConfig'
```

This tells Django to use your `AccountsConfig` class when loading the accounts app. In newer Django versions this is automatic, but it's safe to be explicit.

---

## What the flow looks like now
```
POST /api/accounts/register/
        ↓
register_view runs
        ↓
User.objects.create_user(username, email, password)
        ↓
Django saves the User to DB
        ↓
post_save signal fires automatically
        ↓
create_user_profile() runs
        ↓
created=True → UserProfile.objects.create(user=instance)
        ↓
UserProfile saved to DB linked to that User
        ↓
save_user_profile() runs
        ↓
instance.userprofile.save()
        ↓
Both User and UserProfile exist, fully linked
```

Your register view doesn't need to know about `UserProfile` at all. It just creates the user. Signals handle the rest automatically.
```
